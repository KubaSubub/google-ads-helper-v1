Pisze spec
docs/specs/{slug}.md
  • User Story (perspektywa Kuby)
  • Acceptance Criteria >= 3
  • Edge Cases >= 2
  • Out of Scope (PM decyduje)
  • Success Metric (1 zdanie)
Skip paths
nie kazdy task wymaga spec
